# Example 01: Big O Runtime Comparison

## Problem
Implement and time O(1), O(log n), O(n), O(n log n), O(n²) on the same input size to observe growth rates.

## What You'll Learn
- How runtime grows as n increases
- Why choosing the right algorithm matters

## Expected Output
With n=1000, O(n²) is ~1000x slower than O(n).
